# 📓 NotebookLM Clone

A full-stack AI-powered document intelligence platform — a clone of Google's NotebookLM. Upload any PDF and get instant AI-generated insights: summaries, FAQs, study guides, briefing documents, interactive mind maps, and a streaming RAG chat grounded in your document.

---

## 🗂 Monorepo Structure

```
notebooklm/
├── NotebookLM/          # Backend — Express + TypeScript API
└── notebooklm-clone/    # Frontend — Next.js 15 + Tailwind CSS
```

---

## ✨ Features

### AI Insights
| Feature | Description |
|---|---|
| **Summary** | Concise document overview generated by GPT-4o-mini |
| **Study Guide** | Structured learning material with key concepts |
| **FAQ** | Auto-generated Q&A pairs from document content |
| **Briefing Doc** | Executive-style briefing extracted from the document |
| **Mind Map** | Interactive visual knowledge graph (requires Study Guide first) |

### RAG Chat
- **Streaming chat** grounded in Pinecone vector search over uploaded documents
- Per-message **source citations** with excerpt snippets and similarity scores
- Conversation history sent with every request (last 8 turns, 3 000 chars/turn)
- Server-Sent Events (SSE) for real-time token streaming

### Document Management
- Upload PDF files — auto-generates an AI title and a DALL-E cover image (background job via Agenda.js)
- Dashboard with search + pagination
- Inline note rename (double-click or kebab menu)
- Google Drive file picker integration

### Authentication
- Google OAuth 2.0 via Passport.js
- Session-based auth (express-session + MongoDB store)
- Frontend session validation via `/auth/me`

## 🚀 Getting Started

### Prerequisites
- Node.js ≥ 20
- MongoDB instance (local or Atlas)
- Pinecone account + index
- Google Cloud project with OAuth & Drive API enabled
- OpenAI API key
- Cohere API key

### Backend Setup

```bash
cd NotebookLM
cp .env.example .env
# Fill in all values in .env
npm install
npm run dev
```

Backend runs on `http://localhost:8000` by default.

### Frontend Setup

```bash
cd notebooklm-clone
cp .env.local.example .env.local   # or create manually
npm install
npm run dev
```

Frontend runs on `http://localhost:3000`.

### Environment Variables

#### Backend (`.env`)
```env
PORT=8000
MONGODB_URI=mongodb://localhost:27017/notebooklm
NODE_ENV=development

COOKIE_KEY=your_cookie_secret
JWT_SECRET_KEY=your_jwt_secret
REFRESH_JWT_SECRET_KEY=your_refresh_jwt_secret

# Google OAuth & Drive
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

# AI Keys
```

## 🔄 Key Workflows

### 1. Upload a Document
1. User picks a PDF from the dashboard upload dialog
2. `POST /api/v1/notes` (multipart) → backend saves file, splits text, generates AI title, schedules DALL-E image job
3. Note + Doc records created in MongoDB
4. Dashboard refreshes — new card appears

### 2. Generate an Insight (e.g. Summary)
1. User clicks **Generate** on the Summary tab
2. `PUT /api/v1/notes/summary` `{ userId, noteId }` → backend loads PDF, runs LangChain chain, stores result in Doc
3. UI polls via `useInsight` hook and renders result when `status === 'cached'`
4. Subsequent visits hit `GET /api/v1/notes/summary` → served from DB (no re-generation)

### 3. Mind Map (dependency-gated)
- Requires Study Guide to be generated first (backend uses `doc.studyGuide || doc.briefingDoc` as input)
- Frontend locks the Mind Map tab with a 🔒 until Study Guide status is `cached`

### 4. Chat with Document (RAG)
1. User types a message in the Chat tab → navigates to `/notes/[noteId]/chat`
2. Frontend posts to Next.js API route `/api/notes/chat` → proxied to `POST /api/v1/notes/chat`
3. Backend: embeds query (Cohere) → queries Pinecone (filtered by `noteId`) → streams GPT-4o tokens via SSE
4. After stream: backend sends `sources` event with chunk excerpts and similarity scores
5. Frontend `streamChat()` utility reads the SSE line-by-line and calls `onToken` / `onSources` / `onDone` callbacks

## 🔒 Auth Flow

1. `GET /api/v1/auth/google` → redirect to Google
2. Google redirects to `CALLBACK_URL` → Passport creates/upserts User in MongoDB
3. Session cookie set → frontend `/auth/me` validates session on every load
4. `POST /auth/logout` destroys session

## 📝 Data Models

### User
```
name, email, image, googleId, googleAccessToken, googleRefreshToken, timestamps
```

### Note
```
title, image (DALL-E URL), userId → User, timestamps
```

### Doc
```
title, fileName, description?,
summary?, studyGuide?, briefingDoc?, FAQ?, mindMap? (JSON string),
noteId → Note, userId → User, timestamps
```
